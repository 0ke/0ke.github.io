驱动过滤--串口过滤

无标签
发布日期:   2022-05-20

过滤的概念

过滤是在不影响上层和下层接口的情况下，在Windows系统内核中加入新的层，从而不需要修改上层的软件或者下层的驱动程序，加入新的功能。

过滤的主要方法是对一个设备对象（Device Object）进行绑定。设备对象与实际的硬件绑定，从而完成实际硬件的功能。

串口过滤

首先认为：一个真实的设备对应一个设备对象（实际情况不是这样），通过编程生成一个虚拟的设备对象，并”绑定“ 在一个真实的设备上。绑定之后，本来操作系统发送给真实设备的请求，就会先发送到被绑定的虚拟设备上。

设备绑定的内核API

下面介绍一个实现绑定功能的函数。

NTSTATUS
IoAttachDevice(
    _In_ _When_(return==0, __drv_aliasesMem)
    PDEVICE_OBJECT SourceDevice, //调用者生成的用来过滤的虚拟设备
    _In_  PUNICODE_STRING TargetDevice,//要被绑定的目标设备
    _Out_ PDEVICE_OBJECT *AttachedDevice//用来返回的二级指针，绑定完成后，被绑定的设备指针被返回的地址。
    );

第二个参数TargetDevice并不是PDEVICE_OBJECT，而是一个字符串（UNICODE_STRING），实际上这个字符串是要被绑定的设备的名字。

Windows中许多设备对象是有名字的，但并不是所有的设备对象都有名字。只有有名字的设备，才能用这个内核API进行绑定。

如果一个设备被其他的设备绑定了，它们在一起的一组设备，被称为设备栈（之所以称为栈，是由于和请求的传递方式有关）。实际上，IoAttachDevice总是会绑定设备栈上最顶层的那个设备。

在windows2000下应该使用IoAttachDeviceToDeviceStack。在windows2000以上系统使用IoAttachDeviceToDeviceStacksafe

PDEVICE_OBJECT  //返回类型导致这个函数不能返回一个明确的错误码，但是返回值如果为NULL，则表示绑定失败了。
IoAttachDeviceToDeviceStack(
    _In_ _When_(return!=0, __drv_aliasesMem)
    PDEVICE_OBJECT SourceDevice,
    _In_ PDEVICE_OBJECT TargetDevice
    );
NTSTATUS 
IoAttachDeviceToDeviceStackSafe(
    _In_  PDEVICE_OBJECT SourceDevice,//过滤设备
    _In_  PDEVICE_OBJECT TargetDevice,//要绑定的设备栈中的设备
    _Outptr_ PDEVICE_OBJECT *AttachedToDeviceObject//多出的这个参数用以返回最终被绑定的设备
    );

生成过滤设备并绑定

在绑定设备之前，需要生成一个用于过滤的设备。

NTSTATUS
IoCreateDevice(
    _In_  PDRIVER_OBJECT DriverObject, //这个驱动本身的驱动对象.系统提供，从DriverEntry中传入。
    _In_  ULONG DeviceExtensionSize, //扩展设备，先传入0
    _In_opt_ PUNICODE_STRING DeviceName, //设备名字。传入NULL即可
    _In_  DEVICE_TYPE DeviceType,//设备类型，保持和被绑定的设备类型一致即可。
    _In_  ULONG DeviceCharacteristics,//设备特征，0或者FALSE
    _In_  BOOLEAN Exclusive,
    _Outptr_result_nullonfailure_
    _At_(*DeviceObject,
        __drv_allocatesMem(Mem)
        _When_((((_In_function_class_(DRIVER_INITIALIZE))
               ||(_In_function_class_(DRIVER_DISPATCH)))),
             __drv_aliasesMem))
    PDEVICE_OBJECT *DeviceObject
    );

在绑定一个设备之前，应该把这个设备对象的多个子域设置成和要绑定的目标对象一致，包括标志和特征。

以下函数生成一个设备，然后绑定在另一个设备上。

NTSTATUS ccpAttachDevice(PDRIVER_OBJECT DriverObject,//和驱动入口点驱动对象相同
                         PDEVICE_OBJECT oldobj,
                         PDEVICE_OBJECT *fltobj,
                         PDEVICE_OBJECT *next)
{
    NTSTATUS status;
    PDEVICE_OBJECT topdev = NULL;

    //生成设备，然后绑定

    status = IoCreateDevice(DriverObject,
        0,
        NULL,
        oldobj->DeviceType,
        0,
        FALSE,
        fltobj
        );
    if (status != STATUS_SUCCESS)
    {
        return status;
    }
    //拷贝重要标志位
    if (oldobj->Flags & DO_BUFFERED_IO)
        (*fltobj)->Flags |= DO_BUFFERED_IO;
    if (oldobj->Flags|=DO_DIRECT_IO)
        (*fltobj)->Flags |= DO_DIRECT_IO;

    if (oldobj->Characteristics & FILE_DEVICE_SECURE_OPEN)
        (*fltobj)->Characteristics |= FILE_DEVICE_SECURE_OPEN;
    (*fltobj)->Flags |= DO_POWER_PAGABLE;
    //将一个设备绑定到另一个设备上
    topdev = IoAttachDeviceToDeviceStack(*fltobj, oldobj);
    if (topdev == NULL)
    {
        //如果绑定失败了，需要摧毁设备，返回错误
        IoDeleteDevice(*fltobj);
        *fltobj = NULL;
        status = STATUS_UNSUCCESSFUL;
        return status;
    }
    *next = topdev;
    (*fltobj)->Flags = (*fltobj)->Flags & ~DO_DEVICE_INITIALIZING;
    return STATUS_SUCCESS;
}

从名字获得设备对象

通过一个设备名字，使用函数IoGetDeviceObjectPointer可以获得这个设备对象的指针。

NTSTATUS
IoGetDeviceObjectPointer(
    _In_  PUNICODE_STRING ObjectName,//设备名字
    _In_  ACCESS_MASK DesiredAccess,//期望访问的权限，可以直接填写FILE_ALL_ACCESS
    _Out_ PFILE_OBJECT *FileObject,//返回参数，获得这个对象的同时会得到一个文件对象。对于串口过滤来说，没没什么作用。
    _Out_ PDEVICE_OBJECT *DeviceObject
    );

注意：再使用上述函数之后必须把这个文件对象“解除引用”，不然会引起内存泄漏，发生危险。

要得到的设备对象返回在参数DeviceObject中。示例如下

#include <ntddk.h>
//RtlStringCchPrintfW
#include <ntstrsafe.h>
//为了使用ntstrsafe静态库，这样才能兼容winndows2000
#define NTSTRASFE_LIB
//打开一个端口设备
PDEVICE_OBJECT ccpOpenCom(ULONG id, NTSTATUS *status)
{
    //外面输入的是串口id，这里改写成字符串形式
    UNICODE_STRING name_str;
    static WCHAR name[32] = { 0 };
    PFILE_OBJECT fileobj = NULL;
    PDEVICE_OBJECT devobj = NULL;

    //根据id转换成串口的名字
    memset(name, 0, sizeof(WCHAR)* 32);
    RtlStringCchPrintfW(name, 32, L"\\Device\\Serial%d", id);
    RtlInitUnicodeString(&name_str, name);
    //打开设备对象
    *status = IoGetDeviceObjectPointer(&name_str, FILE_ALL_ACCESS, &fileobj, &devobj);
    //如果打开成功，一定将文件对象解除引用
    if (*status == STATUS_SUCCESS)
    {
        ObDereferenceObject(fileobj);
    }
    //返回设备对象
    return devobj;
}

绑定所有串口

由于不知道计算机上一共有多少串口，所以这里绑定所有串口。

// 设定计算机上只有32个串口
#define CCP_MAX_COM_ID 32
// 保存所有的过滤设备指针
static PDEVICE_OBJECT s_fltobj[CCP_MAX_COM_ID] = { 0 };
// 保存所有真实设备指针
static PDEVICE_OBJECT s_nextobj[CCP_MAX_COM_ID] = { 0 };
//绑定所有串口
VOID ccpAttachAllComs(PDRIVER_OBJECT driver)
{
    PDEVICE_OBJECT com_obj = NULL; //这里记得初始化一下
    NTSTATUS status = STATUS_SUCCESS;

    for (ULONG i = 0; i < CCP_MAX_COM_ID; ++i)
    {
        // 获取串口对象引用
        com_obj = ccpOpenCom(i, &status);
        if (com_obj == NULL)
            continue;

        // 这里绑定，并不管绑定是否成功
        ccpAttachDevice(driver, com_obj, &s_fltobj[i], &s_nextobj[i]);
    }
}

通过上述所有内容，虚拟设备已经绑定了真正的串口设备。

获得实际数据

分析操作系统发送给串口设备的请求。

几个概念

（1）每个驱动程序只有一个驱动对象

（2）每个驱动程序可以生成若干个设备对象，且这些设备对象从属于同一个驱动对象。

（3）若干个设备（他们可以属于不同的驱动）依次绑定形成一个设备栈，总是最顶端的设备先收到请求。

在串口过滤中只关心读写请求，因为我们只关心在串口上流过的数据。

//这里的irpsp称为IRP的栈空间，IoGetCurrentIrpStackLocation用来获取当前的栈空间
PIO_STACK_LOCATION irpsp = IoGetCurrentIrpStackLocation(irp);
            if (irpsp->MajorFunction == IRP_MJ_POWER)
            {
                //如果是写.....
            }

            if (irpsp->MajorFunction == IRP_MJ_WRITE)
            {
                //如果是读.....
            }

对于请求的过滤，最终的结局有三种。当过滤一个请求时，需要用IRP处理函数按照上面三种方法之一进行处理，将这些代码写在一个处理函数当中。

1.请求被允许通过了。

2.请求被否决了。

3.过滤完成了这个请求。

NTSTATUS ccpDispatch(PDEVICE_OBJECT device, PIRP irp)
{
    PIO_STACK_LOCATION irpsp = IoGetCurrentIrpStackLocation(irp);
    NTSTATUS status = STATUS_SUCCESS;
    ULONG i = 0, j = 0;

    // 首先要知道发送给了哪个设备
    for (i = 0; i < CCP_MAX_COM_ID; i++)
    {
        if (s_fltobj[i] == device)
        {
            // 所有的电源操作，全部直接放过
            if (irpsp->MajorFunction == IRP_MJ_POWER)
            {
                // 直接发送，然后返回说已经被处理了
                PoStartNextPowerIrp(irp);
                IoSkipCurrentIrpStackLocation(irp);
                return PoCallDriver(s_nextobj[i], irp);
            }

            // 这里只处理写请求过滤,写请求-获得缓存区及其长度
            // 然后打印
            if (irpsp->MajorFunction == IRP_MJ_WRITE)
            {
                // 先获取长度
                ULONG len = irpsp->Parameters.Write.Length;
                // 然后获取缓存区
                PUCHAR buf = NULL;
                if (irp->MdlAddress != NULL)
                    buf = (PUCHAR)MmGetSystemAddressForMdlSafe(irp->MdlAddress, NormalPagePriority);
                else
                    buf = (PUCHAR)irp->UserBuffer;

                if (buf == NULL)
                    buf = (PUCHAR)irp->AssociatedIrp.SystemBuffer;

                // 打印内容
                for (j = 0; j < len; ++j)
                    DbgPrint("comcap: Send Data: %2x\r\n", buf[j]);
            }

            // 这些请求直接下发执行即可，我们并不禁止或改变它
            IoSkipCurrentIrpStackLocation(irp);
            return IoCallDriver(s_nextobj[i], irp);
        }
    }

    // 如果根本就不在被绑定的设备中，那是有问题的，直接返回参数错误
    irp->IoStatus.Information = 0;
    irp->IoStatus.Status = STATUS_INVALID_PARAMETER;
    IoCompleteRequest(irp, IO_NO_INCREMENT);

    return status;
}
