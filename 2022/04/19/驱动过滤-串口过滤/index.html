驱动过滤--串口过滤

过滤的概念

过滤是在不影响上层和下层接口的情况下，在Windows系统内核中加入新的层，从而不需要修改上层的软件或者下层的驱动程序，加入新的功能。

过滤的主要方法是对一个设备对象（Device Object）进行绑定。设备对象与实际的硬件绑定，从而完成实际硬件的功能。

串口过滤

首先认为：一个真实的设备对应一个设备对象（实际情况不是这样），通过编程生成一个虚拟的设备对象，并”绑定“ 在一个真实的设备上。绑定之后，本来操作系统发送给真实设备的请求，就会先发送到被绑定的虚拟设备上。

设备绑定的内核API

下面介绍一个实现绑定功能的函数。

NTSTATUS
IoAttachDevice(
    _In_ _When_(return==0, __drv_aliasesMem)
    PDEVICE_OBJECT SourceDevice, //调用者生成的用来过滤的虚拟设备
    _In_  PUNICODE_STRING TargetDevice,//要被绑定的目标设备
    _Out_ PDEVICE_OBJECT *AttachedDevice//用来返回的二级指针，绑定完成后，被绑定的设备指针被返回的地址。
    );

第二个参数TargetDevice并不是PDEVICE_OBJECT，而是一个字符串（UNICODE_STRING），实际上这个字符串是要被绑定的设备的名字。

Windows中许多设备对象是有名字的，但并不是所有的设备对象都有名字。只有有名字的设备，才能用这个内核API进行绑定。

如果一个设备被其他的设备绑定了，它们在一起的一组设备，被称为设备栈（之所以称为栈，是由于和请求的传递方式有关）。实际上，IoAttachDevice总是会绑定设备栈上最顶层的那个设备。

在windows2000下应该使用IoAttachDeviceToDeviceStack。在windows2000以上系统使用IoAttachDeviceToDeviceStacksafe

PDEVICE_OBJECT  //返回类型导致这个函数不能返回一个明确的错误码，但是返回值如果为NULL，则表示绑定失败了。
IoAttachDeviceToDeviceStack(
    _In_ _When_(return!=0, __drv_aliasesMem)
    PDEVICE_OBJECT SourceDevice,
    _In_ PDEVICE_OBJECT TargetDevice
    );
NTSTATUS 
IoAttachDeviceToDeviceStackSafe(
    _In_  PDEVICE_OBJECT SourceDevice,//过滤设备
    _In_  PDEVICE_OBJECT TargetDevice,//要绑定的设备栈中的设备
    _Outptr_ PDEVICE_OBJECT *AttachedToDeviceObject//多出的这个参数用以返回最终被绑定的设备
    );

生成过滤设备并绑定

在绑定设备之前，需要生成一个用于过滤的设备。

NTSTATUS
IoCreateDevice(
    _In_  PDRIVER_OBJECT DriverObject, //这个驱动本身的驱动对象.系统提供，从DriverEntry中传入。
    _In_  ULONG DeviceExtensionSize, //扩展设备，先传入0
    _In_opt_ PUNICODE_STRING DeviceName, //设备名字。传入NULL即可
    _In_  DEVICE_TYPE DeviceType,//设备类型，保持和被绑定的设备类型一致即可。
    _In_  ULONG DeviceCharacteristics,//设备特征，0或者FALSE
    _In_  BOOLEAN Exclusive,
    _Outptr_result_nullonfailure_
    _At_(*DeviceObject,
        __drv_allocatesMem(Mem)
        _When_((((_In_function_class_(DRIVER_INITIALIZE))
               ||(_In_function_class_(DRIVER_DISPATCH)))),
             __drv_aliasesMem))
    PDEVICE_OBJECT *DeviceObject
    );

在绑定一个设备之前，应该把这个设备对象的多个子域设置成和要绑定的目标对象一致，包括标志和特征。

从名字获得设备对象

通过一个设备名字，使用函数IoGetDeviceObjectPointer可以获得这个设备对象的指针。

NTSTATUS
IoGetDeviceObjectPointer(
    _In_  PUNICODE_STRING ObjectName,//设备名字
    _In_  ACCESS_MASK DesiredAccess,//期望访问的权限，可以直接填写FILE_ALL_ACCESS
    _Out_ PFILE_OBJECT *FileObject,//返回参数，获得这个对象的同时会得到一个文件对象。对于串口过滤来说，没没什么作用。
    _Out_ PDEVICE_OBJECT *DeviceObject
    );

注意：再使用上述函数之后必须把这个文件对象“解除引用”，不然会引起内存泄漏，发生危险。

要得到的设备对象返回在参数DeviceObject中。示例如下

#include 
//RtlStringCchPrintfW
#include 
//为了使用ntstrsafe静态库，这样才能兼容winndows2000
#define NTSTRASFE_LIB
//打开一个端口设备
PDEVICE_OBJECT ccpOpenCom(ULONG id, NTSTATUS *status)
{
    //外面输入的是串口id，这里改写成字符串形式
    UNICODE_STRING name_str;
    static WCHAR name[32] = { 0 };
    PFILE_OBJECT fileobj = NULL;
    PDEVICE_OBJECT devobj = NULL;

    //根据id转换成串口的名字
    memset(name, 0, sizeof(WCHAR)* 32);
    RtlStringCchPrintfW(name, 32, L"\\Device\\Serial%d", id);
    RtlInitUnicodeString(&name_str, name);
    //打开设备对象
    *status = IoGetDeviceObjectPointer(&name_str, FILE_ALL_ACCESS, &fileobj, &devobj);
    //如果打开成功，一定将文件对象解除引用
    if (*status == STATUS_SUCCESS)
    {
        ObDereferenceObject(fileobj);
    }
    //返回设备对象
    return devobj;
}
